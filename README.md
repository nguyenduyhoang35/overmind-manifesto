# Overmind

### A Knowledge-Driven Development Manifesto

> You don't need AI to execute faster.
> You need AI to execute **correctly**.
> And "correctly" depends entirely on the quality of knowledge you give it.

## Core Idea

**Knowledge determines output quality.** Before asking "how should AI execute?" — ask "what does AI know?"

The same AI, same model — iteration 0 produces 84% quality, iteration 21 produces 100%. The difference comes entirely from accumulated knowledge.

Three things you need:
1. **A knowledge store** — one place where both humans and AI can read all important knowledge
2. **Skills to interact with knowledge** — structured ways to discover, decide, plan, retro, and query
3. **A feedback loop** — every execution feeds back into the knowledge store

One loop: `Knowledge → Execute → Feedback → Knowledge (updated)`

## When NOT to Use This

- Quick prototypes that need validation in a day
- Tasks too simple to warrant the overhead
- You're already a complete expert and can describe everything in one prompt
- Wrong output is cheap to fix

Use it when: **cost of getting it wrong > cost of the knowledge ceremony.**
